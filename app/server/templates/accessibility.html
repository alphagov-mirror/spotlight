Accessibility statement for GOV.UK Performance Platform

This accessibility statement explains how accessible this service is, what to do if you have difficulty using it, and how to report accessibility problems with the service.

This service is part of the wider GOV.UK website. There’s a separate accessibility statement for the main GOV.UK website.

This page only contains information about the GOV.UK Performance Platform service, available at https://www.gov.uk/performance.

We’ve also made the text in the service as simple as possible to understand. AbilityNet has advice on making your device easier to use if you have a disability.

This service is run by the Government Digital Service (GDS). We want as many people as possible to be able to use this service. This means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 300% without the text spilling off the screen
  • get from the start of the service to the end using just a keyboard
  • get from the start of the service to the end using speech recognition software
  • listen to the service using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

How accessible is this website

We know some of this website is not fully accessible; the following accessibility problems have been identified:

  • Broken ARIA reference, this will cause problems for tools which help with accessibility.
  • Lack of Adequate contrast between foreground and background colours which makes it difficult for those with low vision to differentiate between rows in tables, especially users with low vision.
  • Heading level skipped. Headings provide document structure and facilitate keyboard navigation by users of assistive technology.
  • Redundant links on pages which may create additional navigation for keyboard and screen reader users.

We are in the process of transitioning all these platforms to new arrangements, so it would be disproportionate to make the necessary changes to the old platforms.

What to do if you have difficulty using this service

If you need any part of this service in a different format please contact https://www.gov.uk/contact/govuk

We’ll consider your request and get back to you within two working days.

Reporting accessibility problems with this service

If you find any problems that are not listed on this page or think we’re not meeting accessibility requirements, please contact us via the following link https://www.gov.uk/contact/govuk

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you are not happy with how we respond to your complaint, contact the Equality Advisory and Support Service, which is run on behalf of EHRC. Technical information about this service’s accessibility We’re committed to making this service accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This service is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the non-compliances listed below.

Non compliant content

The content listed below is non-accessible for the following reasons.

  • Broken ARIA reference, 1.3.1 Info and Relationships (Level A) and 4.1.2 Name, Role, Value (Level A)
  • some foreground and background colours lack contrast, so users with low vision for example they may will find it difficult to pick out rows in tables - this fails WCAG 2.1 success criterion 1.4.3 (Contrast)
  • some pages are missing a heading - this fails WCAG 2.1 success criterion 1.3.1 (Info and Relationships)
  • Adjacent links go to the same URL. This results in additional navigation and repetition for keyboard and screen reader users. - this fails WCAG 2.1 2.4.4 Link Purpose (In Context) (Level A) success criterion 2.1.1 (Keyboard)

Preparation of this accessibility statement

We last tested this service for compliance with WCAG 2.1 AA on 18 September 2020

We used this approach to deciding on a sample of pages to test.

  • We used Lighthouse to audit each page for accessibility issues.
  • We used Wave to automatically test for accessibility issues on pages that do not need users to sign in.

We tested:

  • Our main website platform, available at www.gov.uk/performance
  • Link to a set of dashboards at www.gov.uk/performance/other
  • A dashboard www.gov.uk/performance/govuk-pay
